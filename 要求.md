# React19 + Antd6 + Vite + Bun 项目搭建要求（工程规范版）

## 1. 文档目标与范围
- 目标：从零搭建一个可长期维护的前端工程骨架，技术基线为 React19、Antd6、Vite、Bun。
- 范围：覆盖初始化、目录结构、核心依赖、工程规范、质量门禁、环境管理、交付约束。
- 非目标：不在本阶段实现业务模块，只定义可落地的工程框架。

## 2. 技术基线（必须项）
- 运行时与包管理：Bun（统一 `bun install`、`bun run`）。
- 构建工具：Vite（React 官方生态首选，冷启动与构建速度优先）。
- UI 组件库：Antd6（与 React19 兼容，提供企业级组件能力）。
- 语言：TypeScript（默认开启严格模式）。
- 路由：React-Router-Dom
- 请求层：Axios（统一拦截器、错误处理、超时策略）。
- 状态管理：优先轻量方案（React Query + Context）；仅在复杂全局状态时引入 Zustand。
- 样式方案：CSS Modules + 少量全局样式（主题变量统一收口）。

## 3. 初始化与脚手架（阶段一）
### 必须项
- 使用 Bun 创建项目并安装依赖，统一锁文件与安装命令。
- 初始化 Git 仓库并提交首个基线版本（含 `.gitignore`、`README.md`）。
- 明确 Node/Bun 最低版本要求，写入 `README.md` 与 `.nvmrc`（如团队使用 nvm）。
- 配置基础脚本：`dev`、`build`、`preview`、`typecheck`、`test`。

### 可选项
- 接入 `commitlint + husky + lint-staged` 做提交前校验。
- 预置 `changeset` 用于版本变更记录（如未来会发布包）。

## 4. 目录结构规范（阶段二）
推荐采用“按业务域 + 共享能力”结构，避免纯技术分层导致目录膨胀。

```text
src/
  app/                  # 应用装配：路由注册、Provider、全局样式
  pages/                # 页面级入口（路由直达）
  shared/               # 共享能力（组件、hooks、utils、types）
  services/             # API 客户端与请求封装
  styles/               # 主题变量、reset、全局样式
  assets/               # 静态资源
  tests/                # 测试工具与公共测试数据
```

### 必须项
- `app/` 只负责“装配”，不放业务逻辑。
- `shared/` 保持单向依赖。
- `services/` 输出稳定接口，不泄漏 UI 层细节。

## 5. 依赖与配置清单（阶段三）
### 核心依赖（必须）
- `react`, `react-dom`
- `antd`
- `react-router-dom`
- `axios`
- `@tanstack/react-query`

### 工程依赖（必须）
- `typescript`
- `vite`, `@vitejs/plugin-react`
- `eslint`, `@typescript-eslint/*`
- `prettier`
- `vitest`, `@testing-library/react`, `@testing-library/jest-dom`, `jsdom`

### 配置文件（必须）
- `tsconfig.json`：开启 strict、路径别名、模块解析统一。
- `vite.config.ts`：别名、环境变量前缀、按需构建配置。
- `eslint.config.*`：统一语法与代码风格规则。
- `prettier.config.*`：统一格式化。
- `.editorconfig`：统一缩进、换行、编码。
- `.env.development`、`.env.test`、`.env.production`：分环境变量。

## 6. 工程质量门禁（阶段四）
### 必须项
- TypeScript 严格模式下 `typecheck` 必须通过。
- 单元测试可运行，关键公共模块具备最小覆盖（请求封装、权限守卫、核心工具函数）。
- ESLint 规则可运行，禁止明显风险模式（未处理 Promise、循环依赖、未使用导出）。
- 约定 PR 最小检查项：功能说明、影响范围、回归点、截图/录屏（UI 变更）。

### 可选项
- 在 CI 中增加 “类型检查 + 测试 + 构建演练” 三段流水线。
- 引入 `size-limit` 或 `rollup-plugin-visualizer` 做包体积跟踪。

## 7. 路由、权限与状态边界（阶段五）
### 必须项
- 路由分层：公共路由、鉴权路由、兜底路由（404）明确分开。
- 权限控制前置到路由守卫或页面入口，不散落到组件深层。
- 服务端数据统一通过 React Query 管理缓存、重试、失效策略。
- UI 临时状态留在组件内；跨页面共享状态再提升到 Context 或 Zustand。

### 可选项
- 为权限模型增加“角色-路由-按钮”映射表，便于后续运营系统扩展。

## 8. Antd6 接入规范（阶段六）
### 必须项
- 在应用根节点统一配置 `ConfigProvider`，集中管理主题 token 与国际化。
- 仅通过一层 `shared/ui` 封装常用二次组件（如表格、弹窗），避免业务中重复粘贴配置。
- 表单统一约定：校验触发时机、错误提示位置、提交态反馈。

### 可选项
- 建立“页面模板组件”（列表页、详情页、弹窗表单页）提升开发一致性。

## 9. 环境与发布约束（阶段七）
### 必须项
- 环境变量命名统一前缀（Vite 约定），严禁明文密钥进入前端仓库。
- 区分运行配置与构建配置，避免把环境判断散落在业务代码。
- 产物发布前至少执行：依赖安装、类型检查、测试、构建演练。

### 可选项
- 增加灰度发布开关与回滚说明，沉淀到 `README.md`。

## 10. 交付物清单（验收标准）
### 必须项
- 一个可启动的 React19 + Antd6 + Vite + Bun 工程。
- 完整目录结构与基础模块占位。
- 可执行脚本与配置文件齐全。
- 至少 1 组示例页面（列表 + 表单）验证框架链路。
- `README.md` 包含：启动方式、脚本说明、目录说明、环境变量说明。

### 可选项
- 首版 CI 配置文件。
- 首版组件封装规范文档（放在 `docs/`）。

---

本要求文档用于“框架搭建阶段”统一标准。业务开发开始后，允许在不破坏上述边界的前提下增量演进。
